Proposta técnica — aguardando aprovação do visual para implementar

⚙️ Regra de seleção automática de paleta

Canal = linkedin (qualquer stage) Marinho
Stage = educar ou atrair (IG/FB) Petróleo
Stage = conectar ou converter (IG/FB) Terracota
Fallback (caso não encaixe em nenhuma regra) Petróleo

🔄 Pipeline de composição (por post)

IA gera post
→
buscarFotoPexels (portrait)
→
sharp .rotate() + resize 4:5
→
determinarPaleta(canal, stage)
→
gerarSVGOverlay(tag, título, cta, paleta)
→
sharp .composite(svgBuffer)
→
salvar /uploads/post-ID.jpg

🛠️ Biblioteca: sharp + SVG overlay Recomendado

  • sharp já está instalado — sem dependência nova
  • librsvg 2.62.3 + pango já embutidos no sharp: renderiza texto, gradientes, formas, tudo via SVG
  • ~150–300ms por composição (rápido, roda no request de geração)
  • Font Inter será embarcada como arquivo em /public/fonts/ para rendering consistente em qualquer servidor
  • Texto longo: função JS quebra em linhas (no máximo 22 caracteres por linha), gera múltiplos <text> no SVG

❌ Alternativas avaliadas e descartadas

  • node-canvas / @napi-rs/canvas — Canvas API mais familiar, mas requer libcairo como dependência nativa: compilação frágil no deploy Descartado
  • Puppeteer (headless Chrome) — mais flexível mas +300MB, startup lento, overkill para composição simples Descartado
  • Bannerbear / Cloudinary — qualidade alta mas externa, paga, viola princípio de tela única Descartado

🌄 Foto de fundo: degradê de contraste Recomendado

  • Degradê semi-transparente (cor de fundo da paleta) sobre a metade inferior da foto — garante legibilidade em QUALQUER foto do Pexels
  • Pexels não tem parâmetro para "áreas limpas" — filtrar por cor dominante é impreciso e adiciona latência extra
  • Combinação aplicada: já buscamos orientation=portrait (foto mais alta, mais espaço vertical), + degradê como seguro
  • As 3 imagens de demo acima usam este método — o resultado já mostra a qualidade final

⚠️ Constraint de publicação

  • Imagens compostas são salvas em /public/uploads/ como os uploads do usuário
  • Ayrshare precisa de uma URL pública para baixar a imagem — em produção com domínio público, funciona direto
  • Em dev local: a prévia aparece no card de aprovação, mas para publicar no Instagram precisa de túnel (ngrok) ou deploy
  • Rota de publicação receberá ajuste para montar URL completa usando NEXTAUTH_URL
